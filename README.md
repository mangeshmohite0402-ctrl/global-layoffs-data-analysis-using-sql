# 🌍 Global Layoffs Data Analysis using SQL
# 📌 Project Overview
This project focuses on analyzing global layoffs data (2020–2023) using SQL. It involves building a complete data pipeline—from raw data cleaning to extracting meaningful insights—using advanced SQL techniques.
# 📊 Dataset
Real-world dataset of 2,361 companies
Covers layoffs from 2020 to 2023
Total layoffs analyzed: 383,159
⚙️ Key Features
🧹 Data Cleaning & Preparation
Removed duplicate records using CTEs and window functions
Standardized inconsistent data:
Dates
Industry names
Country names
Handled missing values using:
Self-joins
Data imputation techniques
🔄 Data Transformation
Built an end-to-end SQL pipeline for:
Data cleaning
Transformation
Analysis
📈 Exploratory Data Analysis (EDA)
Analyzed layoffs across:
Industries
Countries
Time (year/month)
Developed rolling monthly trend analysis to track layoff patterns over 3 years
🔍 Key Insights
📉 Highest layoffs by a company: Amazon (18,150)
📅 Peak year for layoffs: 2023 (125,677 layoffs)
🏭 Most impacted sectors: Consumer industry & Post-IPO companies
📊 Total layoffs analyzed: 383,159
🛠️ Tools & Technologies
SQL (MySQL / PostgreSQL / SQL Server)
Advanced SQL concepts:
CTEs
Window Functions
Joins
Aggregations
🚀 How to Use
Import the dataset into your SQL environment
Run the data cleaning queries
Execute analysis queries to generate insights
📌 Conclusion
This project demonstrates how SQL can be used not just for querying data, but for building a full data analysis pipeline and extracting real-world business insights.
